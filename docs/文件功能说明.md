## 📝 详细说明

### EncodeDecodeMenu.java

**功能概述**：
编码/解码浮动菜单，是插件中功能最丰富的辅助工具。

**主要功能**：
1. **编码/解码**
   - Base64 编码/解码
   - Hex (十六进制) 编码/解码
   - Unicode 编码/解码
   - URL 编码/解码
   - ROT13 编码/解码
   - 二进制编码/解码

2. **哈希计算**
   - MD5 哈希
   - SHA-256 哈希
   - SHA-512 哈希
   - 计算结果自动复制到剪贴板

3. **时间戳转换**
   - 时间戳转日期
   - 日期转时间戳
   - 支持多种日期格式自动识别（yyyy-MM-dd、yyyy/MM/dd 等）

4. **辅助功能**
   - 撤销操作（恢复原始文本）
   - 替换文本（将结果写回编辑器）
   - 自动选中文本填充

5. **界面布局**
   - 分区显示：转换工具区 + 编码/解码区
   - 按钮成对排列（编码/解码）

---

### AICodeAnalysisToolMenu.java

**功能概述**：
AI 代码分析工具菜单，提供全文代码分析功能。

**主要功能**：
1. 在编辑器顶部「编辑」菜单中添加「AI 代码分析」入口
2. 对编辑器中的全部代码进行 AI 分析
3. 委托 `AICodeAnalysisHelper` 处理提示词输入和分析流程
4. 默认提示词：「请分析以下代码」
5. 支持自定义提示词、快速提示词和 Skill 选择

**使用场景**：
- 完整代码审查
- 全面质量评估
- 深度问题诊断

---

### AICodeAnalysisFloatingMenu.java

**功能概述**：
AI 快速分析浮动菜单，提供选中代码片段的快速诊断。

**主要功能**：
1. 在选中文本后的浮动菜单中添加「AI 快速分析」入口
2. 仅分析选中的代码片段
3. 委托 `AICodeAnalysisHelper` 处理提示词输入和分析流程
4. 默认提示词：「请简要分析以下代码」
5. 仅在有文本选中时显示

**使用场景**：
- 特定函数检查
- 代码块诊断
- 可疑代码段分析

---

### AICodeAnalysisHelper.java

**功能概述**：
AI 代码分析辅助类，封装 ToolMenu 和 FloatingMenu 的公共逻辑，消除重复代码。

**主要功能**：
1. **提示词输入对话框**
   - 多行文本输入框，支持自定义分析提示词
   - 快速提示词按钮（最多 8 个），点击追加到输入框
   - Skill 多选框，选中后追加到提示词末尾

2. **AI 分析流程**
   - 实时显示思考过程和正式分析结果
   - 支持取消分析（二次确认）
   - 支持后台分析模式（不阻塞当前界面）

3. **结果展示**
   - 自动清理 Markdown 格式符号，提升纯文本阅读体验
   - 支持复制分析结果到剪贴板
   - 空结果提示对话框

**核心方法**：

| 方法 | 功能 |
|------|------|
| `showPromptInputDialog(pluginUI, code)` | 显示提示词输入对话框 |
| `startAnalysis(pluginUI, code, userPrompt)` | 开始 AI 分析（前台模式） |
| `startBackgroundAnalysis(...)` | 开始 AI 分析（后台模式） |
| `showResultDialog(pluginUI, result)` | 显示分析结果对话框 |
| `cleanMarkdown(markdown)` | 清理 Markdown 格式符号 |
| `showEmptyResultDialog(pluginUI, errorMsg)` | 显示空结果提示 |

### AIHelper.java

**功能概述**：
AI 工具类，封装所有 AI 相关的网络请求和配置管理逻辑。

**核心方法**：

| 方法 | 功能 |
|------|------|
| `getApiUrl(context)` | 获取 API 地址 |
| `getAiModel(context)` | 获取 AI 模型名称 |
| `getApiKey(context)` | 获取 API 密钥 |
| `getPrompt(context)` | 获取全局分析提示词 |
| `getShortPrompt(context)` | 获取简短分析提示词 |
| `getQuickPrompts(context)` | 获取快速提示词列表（JSON 数组） |
| `getSkills(context)` | 获取自定义 Skill 列表（JSON 数组） |
| `setApiUrl(context, url)` | 保存 API 地址 |
| `setAiModel(context, model)` | 保存 AI 模型名称 |
| `setApiKey(context, key)` | 保存 API 密钥 |
| `setPrompt(context, prompt)` | 保存全局分析提示词 |
| `setShortPrompt(context, prompt)` | 保存简短分析提示词 |
| `setQuickPrompts(context, json)` | 保存快速提示词列表 |
| `setSkills(context, json)` | 保存自定义 Skill 列表 |
| `resetToDefault(context)` | 重置为默认配置（含提示词、Skill、快速提示词） |
| `analyzeCodeWithThinking(...)` | AI 分析代码（显示思考过程） |
| `analyzeCodeWithCustomPrompt(...)` | AI 分析代码（自定义提示词） |
| `analyzeCodeWithUserPrompt(...)` | AI 分析代码（用户提示词 + 系统提示词组合） |
| `analyzeCodeWithUserPromptNoUI(...)` | AI 分析代码（后台无 UI 版本） |
| `analyzeCodeWithAI(...)` | AI 分析代码（通用方法，含 UI 流式显示） |
| `analyzeCodeWithAINoUI(...)` | AI 分析代码（通用方法，无 UI） |
| `runOnMainThread(action)` | 在主线程执行任务 |

**多模型 SSE 格式兼容**：
- **格式A - 独立字段型**：DeepSeek/通义千问（delta.reasoning_content）、百度文心一言（thinking + result）、讯飞星火（text[].thought + content）、message 变体
- **格式B - 结构化块型**：Claude（content[] 数组，type:thinking + type:text）、Gemini（parts[] 数组，thought + text）
- **格式C - 简单字段型**：text 字段、content 直接字段

**思考过程标签检测**：
支持检测和分离以下格式的思考过程标签：
- `<think/>...</think/>`（DeepSeek 原生格式）
- `<thinking>...</thinking>`
- `<reasoning>...</reasoning>`
- `[思考]...[/思考]`
- `[Thinking]...[/Thinking]`
- `**思考过程**...**分析结果**`

**默认配置**：
- API 地址：`https://api.kggzs.cn/v1`
- 模型名称：`MT-v1`
- API 密钥：`sk-K1m4b0U2WoorIub7EhbQTIYRFpQhURRXMdIoZBywCEruujOa`
- 全局分析提示词：硬编码在 `DEFAULT_PROMPT` 常量中（安卓逆向分析专家角色）
- 简短分析提示词：「请简要分析以下代码，指出主要问题和改进建议：」
- 默认快速提示词：「分析此代码是否存在混淆或解密情况，指出混淆技术和解密方法」
