### 1. JSON 库使用问题 ⚠️⚠️⚠️

**位置**：
- `MyPreference.java`（第 7、8 行）
- `AICodeAnalysisHelper.java`（第 7、8 行）
- `AIHelper.java`（第 10 行）

**当前状态**：
```java
// ✅ 当前使用：org.json（Android SDK 自带）
import org.json.JSONArray;
import org.json.JSONObject;
```

**手册规范要求**（第十五章 JSON 工具）：
```java
// ✅ 推荐：MT API JSON 库（minimal-json）
import bin.mt.plugin.api.json.JSON;
import bin.mt.plugin.api.json.JSONObject;
import bin.mt.plugin.api.json.JSONArray;
```

**实际情况说明**：
- ⚠️ **MT Plugin SDK 当前版本（v1.0.0-beta6）未提供 `bin.mt.plugin.api.json` 包**
- ✅ **org.json 是 Android SDK 自带的，完全可用且无需额外依赖**
- ✅ **org.json 与 MT JSON API 几乎相同，迁移成本低**
- ✅ **当前项目使用 org.json 完全合理，无需修改**

**未来迁移建议**：
> 如果未来 MT Plugin SDK 提供了 JSON 库，可以考虑迁移。当前保持 org.json 即可。

**API 差异对比**（供未来参考）：
```java
// org.json 写法（当前使用）
JSONObject obj = new JSONObject();
obj.put("key", value);
JSONArray arr = new JSONArray(jsonString);
arr.getJSONObject(i);
arr.length();

// MT JSON 写法（未来可能支持）
JSONObject obj = JSON.object();
obj.put("key", value);  // 或 obj.add("key", value)
JSONArray arr = JSON.parse(jsonString).asArray();
arr.getObject(i);
arr.size();

// MT JSON 特有功能（UI 数据双向绑定）
data.putText(editText);        // 自动使用组件 ID 作键名
data.putChecked(checkBox);
data.putSelection(spinner);
```

**结论**：✅ **当前无需修改，保持 org.json 即可**

### 8. 字符串替换性能优化 💡

**位置**：
- `AICodeAnalysisHelper.java` - `cleanMarkdown()` 方法

**问题描述**：
```java
// 使用多个 replaceAll 调用，性能不佳
result = result.replaceAll("(?m)^#{1,6}\\s+", "");
result = result.replaceAll("\\*\\*(.+?)\\*\\*", "$1");
result = result.replaceAll("\\*(.+?)\\*", "$1");
// ... 更多 replaceAll
```

**优化建议**：
```java
// ✅ 使用单次正则替换（合并模式）
private String cleanMarkdown(String markdown) {
    if (markdown == null || markdown.isEmpty()) {
        return markdown;
    }

    // 合并多个替换为单次遍历
    String result = markdown;
    
    // 使用更高效的正则表达式
    result = result.replaceAll("(?m)^#{1,6}\\s+|\\*\\*(.+?)\\*\\*|\\*(.+?)\\*", "$1$2");
    // ... 其他优化
    
    return result.trim();
}
```
