## 🎯 @NonNull 注解的作用

### 1. 编译时检查
- 确保方法不会返回 `null`
- 避免调用方出现 `NullPointerException`
- 提高代码安全性

### 2. IDE 支持
- Android Studio/IntelliJ IDEA 会进行空值检查警告
- 提供更好的代码补全和提示
- 帮助开发者理解API契约

### 3. 文档作用
- 明确表达方法的返回值约定
- 其他开发者可以清楚知道方法不会返回 `null`
- 提升代码可读性

### 4. 符合MT规范
- MT插件开发API速查手册明确要求使用 `@NonNull`
- 与MT SDK内部实现保持一致
- 提升插件质量和兼容性

## 📚 相关规范

### MT插件开发API速查手册 - 第十一章 11.1

```java
// 示例：TextEditorFloatingMenu
public class MyFloatingMenu extends BaseTextEditorFloatingMenu {
    @NonNull
    @Override
    public String name() {
        return "功能名称";
    }

    @NonNull
    @Override
    public Drawable icon() {
        return MaterialIcons.get("icon_name");
    }

    @Override
    public boolean checkVisible(@NonNull TextEditor editor) {
        return true;
    }

    @Override
    public void onMenuClick(@NonNull PluginUI pluginUI, @NonNull TextEditor editor) {
        // 实现功能
    }
}
```

**关键点**：
- `name()` 方法必须使用 `@NonNull` 注解
- `icon()` 方法必须使用 `@NonNull` 注解
- 方法参数中的 `TextEditor` 和 `PluginUI` 也应使用 `@NonNull`
