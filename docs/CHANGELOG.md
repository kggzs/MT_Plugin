# 更新日志

## v2.0.3 - 2026-05-20

### 问题修复

- 🔧 **Unicode 解码修复** - 修复使用 split 逻辑时首段文本被静默丢弃的问题，改用正则匹配器
- 🔧 **时区一致性修复** - 修复日期解析和时间戳格式化使用设备默认时区的问题，统一使用 UTC 时区
- 🔧 **Hex 解码验证** - 修复 Hex 解码缺少奇数长度输入验证导致的 `StringIndexOutOfBoundsException`
- 🔧 **HTTP 连接泄漏** - 修复 AIHelper 中 HTTP 连接在异常时未释放的问题，改用 try-with-resources
- 🔧 **空指针异常修复** - 修复 `getErrorStream()` 可能返回 null 导致的 NPE
- 🔧 **多语言支持修复** - 修复 AICodeAnalysisFloatingMenu 和 MyPreference 中 6 处硬编码中文破坏多语言支持

### 代码优化

- 🔧 移除 AIHelper 中重复的正则替换死代码（同一正则连续执行两次）
- 🔧 移除 EncodeDecodeMenu 中无用的 `copy_success` 抽象方法
- 🔧 将 `handleUndo` 重命名为 `handleResetInput`，更准确反映实际行为
- 🔧 优化时间戳阈值判断，从 13 位数改为 10 位数，提升 2286 年后的兼容性

### 多语言更新

- 🌐 为全部 10 种语言添加 5 个新增语言键：
  - `please_select_code_first` - 请先选择代码
  - `reset_input` / `reset_input_success` - 重置输入相关
  - `decode_error_invalid_hex` - 无效十六进制格式错误
  - `load_quick_prompt_data_failed` - 加载快速提示词数据失败

### 技术细节

- 修改 `EncodeDecodeMenu.java`: Unicode 解码改用 `Matcher`，Hex 解码增加长度验证，时区统一 UTC
- 修改 `AIHelper.java`: 使用 try-with-resources 管理连接，增加 getErrorStream null 检查，移除死代码
- 修改 `AICodeAnalysisFloatingMenu.java`: 硬编码中文替换为资源键
- 修改 `MyPreference.java`: 3 处硬编码中文替换为资源键
- 更新全部 10 个语言包文件 (`strings*.mtl`)

## v2.0.2 (当前版本) - 2026-05-20

### 新增功能

- 🌐 **多语言支持** - 支持10种语言界面
  - 中文（默认）、繁体中文、日语、韩语
  - 阿拉伯语、德语、西班牙语、法语、俄语、英文
- 🔧 **AI 分析后台运行** - 支持后台分析模式，完成后弹出结果
- 🔧 **快速提示词** - 新增快速提示词功能，最多配置10条
- 🔧 **自定义 Skill** - 新增自定义 Skill 功能，可追加到提示词末尾
- 🔧 **时间格式多样化** - 时间插入支持多种格式
  - 公历格式：标准中文、ISO、斜杠、紧凑、带星期
  - 农历格式：传统汉字、农历简写、干支纪日、农历+阿拉伯数字、公农历并列
- 🔧 **赞助功能** - 设置界面支持赞助二维码（微信/支付宝）

### 优化改进

- 🔧 AI 分析辅助类封装，消除重复代码
- 🔧 Markdown 格式清理，提升纯文本阅读体验
- 🔧 取消分析二次确认，防止误操作
- 🔧 API 密钥遮蔽显示，保护隐私

### 技术细节

- 新增 `AICodeAnalysisHelper.java` 封装公共逻辑
- 新增 `TimeFormatHelper.java` 时间格式配置
- 新增 `LunarCalendar.java` 农历计算工具
- 支持10种语言的语言包文件

## 升级指南

### 从 v2.0.1 升级到 v2.0.2

1. 下载最新版本的 `.mtp` 文件
2. 在 MT 管理器中安装新版本
3. 配置会自动保留，无需重新设置

### 从 v2.0 升级到 v2.0.1

1. 下载 v2.0.1 版本的 `.mtp` 文件
2. 在 MT 管理器中安装新版本
3. 如需使用自定义 AI 配置，请在设置中配置

### 从 v1.0 升级到 v2.0

1. 下载 v2.0 版本的 `.mtp` 文件
2. 在 MT 管理器中安装新版本
3. 所有新功能立即可用
